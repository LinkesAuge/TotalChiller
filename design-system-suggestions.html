Game-Themed Buttons

The .button.leather variant exists but is rarely used. The button asset category has game-style button textures that could create a compelling "forged" CTA button for primary actions. Combined with effect assets for hover glow, this creates a strong action hierarchy.

Forged Primary CTA Button High Impact

Create a .button.forged variant for the most important actions (Create Event, New Post, Send Message). This uses a leather/metal texture background with embossed text and a gold glow on hover.

Current .button.primary
Suggested .button.forged

The forged button uses the heading font (Cinzel), uppercase text, a beveled edge effect, and a subtle inner highlight. On hover, a gold glow radiates outward. This provides a clear visual hierarchy where the primary action dominates.

hybrid render button, effect components.css

Surface Hierarchy Tokens

This aligns with the P2.1 backlog item that's still pending. The current surface system has --color-surface and --color-surface-2, but cards, panels, and active cards all look very similar. Define explicit elevation tiers with progressively warmer gold tinting and shadow depth.

Four-Tier Elevation System High Impact

Define four surface levels that create clear visual depth:

Base
Page background
Raised
Panels, filters
Card
Primary content
Elevated
Modals, dropdowns

Add these as new CSS custom properties in theme.css:

--surface-base: rgba(8, 13, 20, 0.9);
--surface-raised: rgba(14, 26, 40, 0.7);
--surface-card: linear-gradient(180deg, rgba(22, 44, 66, 0.9), rgba(14, 30, 45, 0.92));
--surface-elevated: linear-gradient(180deg, rgba(30, 50, 72, 0.95), rgba(18, 36, 54, 0.97));
CSS only theme.css, components.css, layout.css

Typography Refinements

The Cinzel + Inter pairing is strong and appropriate for the theme. A few refinements would improve readability and hierarchy across all surfaces.

Fluid Type Scale Medium Impact

The current html { font-size: 15px; } is slightly below the recommended 16px minimum for body text on mobile. Adopt a fluid base using clamp() so it scales gracefully from small to large viewports:

html { font-size: clamp(15px, 0.3vw + 14.5px, 16px); }

Also add fluid heading sizes for better scaling on tablets:

.hero-title { font-size: clamp(1.2rem, 3vw + 0.5rem, 1.6rem); }
.top-bar-title { font-size: clamp(0.92rem, 1vw + 0.7rem, 1.12rem); }
.news-card-title { font-size: clamp(1.1rem, 2vw + 0.5rem, 1.5rem); }
CSS only theme.css, components.css, news.css
Heading Weight Hierarchy Low Impact

Cinzel is used uniformly at weight 700 across most headings. Create a weight hierarchy for visual depth:

Page Title (900)
Card Header (700)
Section Label (600)
Meta/Subtitle (400)

This four-level weight scale creates a clear visual hierarchy while staying within a single font family. Reserve weight 900 for page-level titles only, 700 for card headers, 600 for section labels, and 400 for subtle meta text.
